Надо написать на Golang утилиту, которая отслеживает изменения файлов и каталогов (создание, удаление, изменение) по указанному пути.

Программа состоит из нескольких программых частей.

## 1. Мониторинг каталога  
1. Каталог для мониторинга (RootFolder) долджен задаваться командной строкой, но если он не задан - то берем из константы.   
2. Для отслеживания изменений на диске используем библиотеку fswatcher.
3. Cooldown изменений - 300 ms.
4. Изменения возвращаются событием в канал в виде массива структур вида: 
    FileChange{
	   time time.Time // время изменения
	   changeType FileChangeType// тип изменения - смотри ниже описание типа FileChangeType в пункте 5.
	   isFile bool // если изменения для файла то True, если для каталога - то False.   
	   name // поле заполняется именем файла, если изменился файл.    
	   size // размер файла, если файл был Created или Modified. Если было изменение каталога (когда isFile=false), то размер равен 0.
	   fullPath // полный путь до измененного файла или каталога, относительно RootFolder.  
	   }
	  
5. Описание типа изменения:	  
type FileChangeType int
const (
    None  FileChangeType = iota // 0
    Created                 // 1
    Modified               // 2
	Renamed               // 3
    Removed                // 4
)
	   
	   
## 2. Накопление данных о файлах в каталоге 

### 2.1 Структура описания файлов в памяти. 

Мы имеем структуру с информацией о всех файлах в каталоге.  
Структура иерархична и описывает расположение файлов на диске.
Примерная структура: 

DiskItemInfo
{
  name
  isFile bool
  size // размер файла, или 0 для каталога
  items[] DiskItemInfo // массив (срез) вложенных элементов
  
  changeType  ChangeType
  changeTime  time.Time
}
 
Когда приложение стартует оно анализирует каталог и заполняет иерархию  DiskItemInfo согласно расположению файлов на диске.  
Значение changeType имеет значение "None", а changeTime имееи "zero value".

Root элемент - это каталог RootFolder, и его items - это подкаталогии файлы. Каждый каталог имеет свои подкаталоги и файлы.


### 2.2 Обновление структуры с описанием файлов.

1.  Когда получаем событие об изменении файлов на диске, то находим соотвествующие элементы в структуре DiskItemInfo по пути файла или каталога. Если элемент не найден, то добавляем его в структуру согласно его пути на диске.   

2. Для каждого измененного элемента берем значения из FileChange и заполняем поля в DiskItemInfo.


## 3. Отрисовка структуры с описанием файлов на экране

1. Для отрисовки на экране используем библиотеку tcell.  
2. Проходим иерархически по всем элементам дерева DiskItemInfo и отрисовываем каждый Item с отступом, соответствующим уровню вложенности. Если это файл (IsFile=true), то дополнительно уазываем размер файла. Если это каталог, то после названия выводим символ "/". Каталоги выводятся первыми, затем файлы каждого каталога.

Пример вывода дерева файлов:  

/somefolder/
|+SubFolder1/
|+SubFolder2/  
| |-SubFolder2-1\
| | |-file2-1-1.jpg  34 kb
| | |-file2-1-2.jpg  100 kb
| |-file1.txt 30 b
| |-file2.txt 3 Mb
| |-file3.txt 150 kb
